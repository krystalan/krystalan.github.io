1.数据仓库基本概念

1.1 什么是数据仓库（Data Warehouse）

“A data warehouse is a subject-oriented, integrated, time-variant, and nonvolatile（面向主题的、集成的、时变的、非易失的） collection of data in support of management’s decision-making process“.（—W.H.Inmon）

是用来做决策的数据库而不是用于运营的。
数据仓库建模（Data Warehousing）：构建并使用数据仓库的过程

1.2 OLTP&OLAP

OLTP：On-Line Transaction Processing，联机事务处理
OLAP：On-Line analytical processing，联机分析处理

1.3 为什么DW要是独立的？

说白了就是因为OLAP和OLTP的不同，数据库主要是为了OLTP服务的，DW是为了OLAP服务的。
（1）DW中需要做一些复杂的OLAP查询，多维视角观察并综合分析数据。它不用考虑OLTP要考虑的什么高并发啊多线程啊之类的。
（2）DW需要历史数据做分析，DB可以不要，DB我只管运营！
（3）DW需要从不同的数据源去做一个异构的融合，这些异构可能是编码不同，格式不同，叫法不同等等。
注意：现在也有许多系统直接在关系型数据库上做OLAP。

1.4 数据仓库的多层结构


分为三部分，首先是数据源，中间是数据存储，接着就是OLAP，最高层则是最终设计出来的工具。

1.5 数据仓库建模的三种方式

（1）Enterprise Warehouse
在整个组织内收集全部的主题下的信息
（2）Data Mart
数据集市，每个集市是在公司范围内对特定的用户组有价值的数据子集。它的范围仅限于特定的、选定的集团。例如销售组。
（3）Virtual Warehouse
虚拟仓库，没有直接去构建一个数据仓库，而是从关系型数据库上用视表直接看。

1.6 ETL

（1）Data Extraction
从多样的、异质的外部数据源获取数据
（2）Data Cleaning
检测数据错误并修正或清除
（3）Data transformation
按需求转换数据格式
（4）Load
排序、汇总、合并、计算视图、检查完整性，并构建指示和分区
（5）Refresh
从数据源中更新数据

1.7 元数据存储库

元数据是定义数据仓库对象的数据

2.3 数据立方体度量：三个类别

（1）分布性度量：
如果对n个聚合值应用该函数所得到的结果与对所有数据应用该函数而不进行分区得到的结果相同。例如：count()、sum()、min()、max()
（2）代数度量：
如果它可以由一个有M个参数的代数函数来计算（其中M是一个有界整数），每一个参数都是通过应用一个分配聚合函数得到的。例如：avg()、min_N()、standard_deviation()
（3）整体度量：
在描述子聚合所需的存储大小上没有固定的界限。例如：median()、mode()、rank()

3.数据仓库的设计和使用

3.1 关于数据仓库设计的四个视图

（1）自上而下视图
允许选择数据仓库所需的相关信息，即先在一个总体的角度去考虑需要构建什么样的数据库和相关信息，在具体到每个子任务。
（2）数据源视图
公开操作系统捕获、存储和管理的信息，即数据源有哪些数据，反过来可以帮助我们更好地设计仓库。
（3）数据仓库视图
直接站在DM的角度去考虑，由事实表和维度表所构成。
（4）业务查询视图
从最终用户的角度查看数据仓库中的数据透视图。

3.2 数据仓库设计步骤

（1）自上而下或自下而上
（2）从软工视角：瀑布式与螺旋式
（3）传统数据仓库的设计过程
选择商业流程→选择数据原子级别→选择维度→选择衡量（measure）

3.3 数据仓库的使用

（1）处理信息
支持查询、基本的统计分析、对于表图的支撑和汇报。
（2）数据分析
对数据仓库的数据进行多维度分析并支撑基本的OLAP操作
（3）数据挖掘
从隐藏模式中发觉知识、挖掘关联规则、构建分析模型去做分类聚类或预测等

4.3 OLAP服务框架

（1）Relational OLAP(ROLAP)
使用关系型或者扩展关系型数据库管理系统并管理仓库中的数据。

基于稀疏的多维存储引擎。
