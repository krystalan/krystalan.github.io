2.马尔可夫奖励过程MRP

2.1 基本概念

马尔可夫奖励过程 = 马尔可夫链 + 奖励
奖励函数是一个期望，当你到达某个状态的时候，可以获得多大的奖励。

如上图所示（s1状态有5个奖励，s7状态有10个奖励），对于马尔可夫链的每一个状态去定义一个奖励。

其余概念：
（1）Horizon：每个episode里最大的时间步数，可以是无限的，当有限时，整个MRP称为有限马尔可夫（奖励）过程。
（2）Return：把每一步的奖励进行折扣，获得总体收益，越往后得到的奖励折扣的越多。

说明更期望得到现有的奖励，未来得到的奖励就要打折扣。
γ为discount factor折价因子。
（3）state value function：
对于马尔可夫奖励过程定义一个状态价值函数（关于return的期望）

代表从当前的状态开始，有可能获得的价值期望，可以反映出当前状态的好坏。

如本节一开始的例子，我们现在来计算s4状态的价值函数。
可以通过MC采样过程计算，也可以使用贝尔曼等式来计算：

基本思路与PageRank相似，通过迭代至参数收敛即可。
迭代方法：动态规划、蒙特卡罗、TD learning
①来看一下蒙特卡罗方法：

简述一下就是：从t时间下的s状态开始，产生N条轨迹，计算这N条轨迹的奖励的平均值，即为s状态下的价值。
蒙特卡罗的思想非常简单：就是通过大量采样近似求解，其一个非常经典的应用就是计算积分：在空间上计算积分区域内的采样点占比。
②直接迭代计算：当两次迭代中奖励变化小于某阈值时停止迭代，和EM算法类似。

如果MRP状态少的话，可以把该等式写成矩阵形式，但写成矩阵形式之后，我们无法保证矩阵满秩，所以并不能直接求解，再者矩阵求逆复杂度极高。（故该方法只适用于状态数少的MRP）

2.3 折价因此γ的用处

（1）避免在有限马尔可夫过程中产生无穷的奖励（有限马尔可夫过程成环）
（2）期望得到尽可能快的奖励，比如相比于一周之后得到100块钱，我更期望现在就得到这100块钱。
（3）人和动物的行为里面，大家都希望得到立刻奖励。
（4）如果γ为0，则只关注当前的奖励；如果γ为1，代表对未来并没有折扣（即未来获得的奖励和当前获得的奖励的价值是一样的）。
（5）是强化学习中的超参数。

3.马尔可夫决策过程MDP

3.1 基本概念

MDP = MRP + Decision = MP + Reward + Decision

可以看出，在MDP中，状态转移和奖励函数需要考虑当前时间agent采取的行为（即decision）。
所以MDP是由状态集合、行为集合、状态转移、奖励函数、决策函数以及折价因子构成的。
